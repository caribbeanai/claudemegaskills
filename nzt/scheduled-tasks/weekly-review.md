# NZT Scheduled Tasks - Weekly Review

> Weekly cognitive maintenance. Like a weekly planning session, but your second brain does the heavy lifting.

## Weekly Review Protocol

**Trigger**: First session of the week, or user requests "/weekly-review"
**Priority**: P2
**Duration**: ~10 minutes of NZT processing

### Phase 1: Retrospective

```yaml
review_last_week:
  - Total tasks completed
  - Tasks carried over (why?)
  - Biggest wins
  - Biggest challenges
  - Time distribution by task type
  - Focus quality assessment
  - Token efficiency trends
```

### Phase 2: Pattern Analysis

```yaml
analyze_patterns:
  - What went well? (repeat these patterns)
  - What didn't work? (avoid or fix)
  - New workflows discovered
  - Automation opportunities identified
  - Skills growing or emerging
  - Areas where NZT could have helped more
```

### Phase 3: NZT Self-Assessment

```yaml
nzt_self_review:
  - Was autonomy level appropriate?
  - Were focus interventions helpful?
  - Did token management work well?
  - Were proactive suggestions accepted?
  - Any corrections or overrides to learn from?
  - SOC protocols that need updating
```

### Phase 4: Planning

```yaml
plan_next_week:
  - Roll over incomplete tasks
  - Set priority for the week
  - Identify potential blockers
  - Schedule deep work blocks
  - Queue background tasks
  - Set improvement goals for NZT
```

### Phase 5: Document Updates

```yaml
update_documents:
  - user-profile.md: Refresh with latest patterns
  - behavior-profiles.md: Log weekly trends
  - self-improvement.md: Record improvements applied
  - SOC docs: Apply any process refinements
  - Commit all changes with review summary
```

## Weekly Review Template

```markdown
# NZT Weekly Review - [Week of Date]

## Last Week at a Glance
- **Tasks Completed**: X
- **Tasks Carried Over**: X
- **Focus Score**: X/10
- **Top Achievement**: [highlight]

## Wins
- [win 1]
- [win 2]

## Challenges
- [challenge 1 + what we learned]

## NZT Performance
- Autonomy Level: [current] (appropriate? adjust?)
- Proactive Suggestion Acceptance: X%
- Token Efficiency: [trend]

## This Week's Plan
1. **Priority 1**: [task/goal]
2. **Priority 2**: [task/goal]
3. **Priority 3**: [task/goal]

## NZT Improvements Applied
- [improvement 1]
- [improvement 2]
```

## Monthly Rollup

On the first weekly review of each month, NZT also generates:

```yaml
monthly_rollup:
  - Month-over-month productivity trends
  - Skill development trajectory
  - NZT autonomy level progression
  - Major accomplishments
  - Patterns that emerged this month
  - Goals for next month
```
